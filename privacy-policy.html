WebWright Privacy Policy

Last updated: April 16, 2026

Overview

WebWright is a browser extension that uses AI to help you interact with web pages. Your privacy matters — this policy explains exactly what data the extension handles and how.

The short version: WebWright stores all your data locally on your device. It only sends data to the LLM provider you choose and configure. We do not operate any servers, collect any analytics, or have access to your data.

Data Stored Locally

The following data is stored in your browser's local storage (chrome.storage.local) and never leaves your device unless you explicitly trigger an action that requires an LLM call:

  • Settings & API Keys — your chosen LLM provider, endpoint URLs, API keys, and model preferences.
  • Personal Info — name, address, and custom fields you optionally save for form-filling. Only used when you ask the agent to fill a form.
  • Workflows — recorded browser action sequences (clicks, typing, navigation) that you save for replay.
  • Research Reports — summaries generated from web research, stored locally for your review.
  • Chat History — conversation messages within the current session only. Not persisted across sessions.

Data Sent to External Services

WebWright sends data to third-party LLM providers only when you initiate an action (chat, agent task, or research). The data sent includes:

  • Page content — text extracted from the current web page (for chat and agent modes).
  • Screenshots — page screenshots sent to vision-capable models when the agent escalates to visual understanding.
  • Your prompt/goal — the text you type to instruct the agent.
  • Personal info — only when you explicitly ask the agent to fill a form, and only the fields relevant to that form.

Data is sent to whichever provider you configure in Settings:

  • Ollama Cloud (ollama.com)
  • Ollama Local (your own machine, localhost)
  • OpenAI (api.openai.com)
  • Anthropic / Claude (api.anthropic.com)
  • Google Gemini (generativelanguage.googleapis.com)
  • DeepSeek (api.deepseek.com)
  • xAI / Grok (api.x.ai)
  • Custom endpoint (user-defined)

Each provider has its own privacy policy governing how they handle data you send. WebWright does not control or have access to data once it reaches the provider's API.

Data We Do NOT Collect

  • No analytics or telemetry
  • No tracking pixels or fingerprinting
  • No browsing history collection
  • No data sent to WebWright developers or any first-party server
  • No advertising or third-party data sharing

Permissions Explained

  • activeTab / tabs — to read and interact with the current page when you give the agent a task. Used to identify the active tab, capture screenshots of pages you ask the agent or chat to operate on, and inject the content script that extracts page elements.
  • scripting — to inject the content script that extracts interactive page elements (buttons, links, inputs, dropdowns) and executes the actions chosen by the agent (click, type, scroll, navigate). Only runs in response to a user-initiated task.
  • storage — to save your settings, API keys, personal info, recorded workflows, and research reports locally via chrome.storage.local. Nothing in storage is transmitted to any first-party server.
  • sidePanel — to display the WebWright sidebar interface, which is the only UI surface the extension uses.
  • webNavigation — to detect single-page app navigations (URL changes without a full reload) so the agent knows when a page has changed and can re-extract interactive elements.
  • debugger — used only as a last-resort fallback in Agent Mode (Tier 4 vision escalation). When DOM-based clicks repeatedly fail to interact with an element, the extension synthesizes a raw coordinate click on the user's active tab via the Chrome DevTools Protocol so the user's task can complete. The debugger is attached on demand, scoped to a single tab, used only for synthetic Input.dispatchMouseEvent commands, and detached as soon as the action finishes. It is never used for network inspection, traffic interception, credential harvesting, background monitoring, or any activity outside the user's currently-running agent task.
  • Host permissions (<all_urls>) — required because the agent can navigate to and interact with any website you direct it to. The user — not the developer — decides which sites the agent visits at runtime by typing a goal. There is no pre-defined target list, so a narrow host pattern would prevent the extension from doing the very thing the user installed it for. No data is read or sent without an explicit user prompt; no background scraping or telemetry occurs.

Single Purpose

WebWright has a single purpose: to act as an autonomous AI agent that performs user-instructed tasks on web pages (navigating, interacting with elements, filling forms, summarizing content, and conducting user-requested research). All permissions above exist exclusively to support that single purpose.

Remote Code Use

WebWright does not load or execute any remote code. All extension JavaScript is bundled in the published package. The only network calls made are to the LLM provider you configure (e.g. Ollama, OpenAI, Anthropic, Google, DeepSeek, xAI) for inference, plus normal page loads performed by the user's browser when the agent navigates on the user's behalf.

Your Controls

  • You can delete all stored data by removing the extension or clearing extension storage.
  • You can use Ollama Local to keep all LLM processing on your own machine with zero external data transfer.
  • Personal info fields are optional — you don't have to fill them.
  • You can delete individual research reports and workflows from the sidebar.
